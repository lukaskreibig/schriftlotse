# Mitwirken

1. Einen kleinen, klaren Branch verwenden.
2. `uv sync --extra dev` ausführen.
3. Ruff, Mypy und Pytest vor dem Pull Request starten.
4. Neue Modelle nur mit Quelle, Lizenz, fixer Revision und reproduzierbarem Qualitätsvergleich
   in die Registry aufnehmen.
5. Keine privaten Scans, Transkriptionen, API-Schlüssel oder Modellgewichte committen.

Fehlerberichte sollen Betriebssystem, Python-Version, Dokumentart und anonymisierte technische
Meldungen enthalten. Dokumentausschnitte nur dann anhängen, wenn Veröffentlichung und Lizenz
eindeutig erlaubt sind.
